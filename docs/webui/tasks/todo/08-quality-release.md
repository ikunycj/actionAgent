# 08. Quality And Release

## 目标
完成 WebUI 的质量门禁、联调验证和发布前检查，确保首发闭环可交付。

## 前置依赖
1. 01-07 阶段已基本完成。
2. Core 侧首发接口具备联调条件。

## 交付物
1. 类型检查、单元测试、组件测试、E2E 测试基线
2. 本地联调清单
3. 发布前验收清单

## 任务清单
- [ ] 8.1 补齐核心路由与组件的单元测试。
- [ ] 8.2 补齐连接、登录、配置、聊天、历史、任务中心的关键集成测试。
- [ ] 8.3 建立 Playwright 首发主链路用例，覆盖连接 Core、登录、配置 Core 级关键参数、发起对话、查看历史、查看任务详情。
- [ ] 8.4 校验各页面的加载态、空态、错误态和权限态。
- [ ] 8.5 完成本地开发环境和生产部署环境的变量检查。
- [ ] 8.6 验证浏览器直连 Core 场景下的 CORS 和 HTTPS 假设。
- [ ] 8.7 输出发布前 smoke checklist。
- [ ] 8.8 明确已知限制与未完成项，并回写相关文档。

## 验收标准
1. 首发主链路具备自动化测试覆盖。
2. 构建产物可被独立部署。
3. 核心页面在典型失败场景下有可接受的反馈。
4. 已知后端依赖和前端限制已文档化，不以口头约定存在。
