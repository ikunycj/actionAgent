# 05. Chat

## 目标
完成基于 `/v1/responses` 的 P0 对话页，包括流式与非流式结果展示，以及运行状态反馈。

## 前置依赖
1. Foundation 阶段完成。
2. Connection And Auth 阶段完成受保护访问和鉴权注入。
3. Core 对话接口可用：`POST /v1/responses`

## 交付物
1. `/app/chat`
2. `/app/chat/:sessionKey`
3. Responses 流式解析器
4. 聊天输入与结果展示组件

## 任务清单
- [ ] 5.1 设计前端消息模型，区分用户消息、临时 assistant 消息和终态消息。
- [ ] 5.2 建立 `session_key` 生成、复用和切换规则。
- [ ] 5.3 接入 `/v1/responses` 的非流式调用。
- [ ] 5.4 接入 `/v1/responses` 的流式调用。
- [ ] 5.5 实现 `fetch + ReadableStream` 流式解析器。
- [ ] 5.6 将流式片段先写入临时消息，再在完成后收敛为终态结果。
- [ ] 5.7 展示 `task_id`、`run_id`、`state`、错误信息和运行反馈。
- [ ] 5.8 防止同一轮发送过程中的重复提交。
- [ ] 5.9 提供从聊天结果跳转到任务详情的入口。
- [ ] 5.10 为非流式、流式、失败、超时场景补齐交互状态。

## 验收标准
1. 用户可在聊天页发起一次完整对话并看到结果。
2. 流式响应能稳定增量展示，不闪烁、不重复。
3. 失败场景下能看到明确错误反馈。
4. 会话 key 与聊天页面状态保持一致。

1. 本阶段只完成“发送与结果展示”，历史读取闭环放在 `06-history.md`。
