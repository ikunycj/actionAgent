# 07. Task Center

## 目标
完成任务列表与任务详情抽屉，为首发闭环补齐任务查看能力。

## 前置依赖
1. Foundation 阶段完成。
2. 现有 bridge 能力可用：
   - `POST /ws/frame`
   - `task.list`
   - `task.get`

## 交付物
1. `/app/tasks`
2. `task.list` 的 bridge 查询封装
3. `task.get` 的详情查询封装
4. 任务详情抽屉

## 任务清单
- [x] 7.1 封装 `/ws/frame` bridge client。
- [x] 7.2 实现 `task.list` queryFn。
- [x] 7.3 实现 `task.get` queryFn。
- [x] 7.4 建立任务列表表格，展示状态、开始时间、完成时间、结果摘要。
- [x] 7.5 建立任务详情抽屉，展示 `task_id`、`run_id`、`state`、`payload`、错误信息。
- [x] 7.6 对非终态任务增加轮询或手动刷新策略。
- [x] 7.7 支持从聊天页和历史页打开对应任务详情。
- [x] 7.8 统一任务状态徽标与时间展示格式。

## 验收标准
1. 用户可查看最近任务列表。
2. 用户可打开详情抽屉查看单个任务详情。
3. 列表默认按终态时间倒序。
4. 任务详情可从其他功能页面联动打开。

## 备注
1. 本阶段先不要求复杂筛选器和高级检索。
2. 若后端未来补充 HTTP 风格任务接口，本阶段成果应只需替换 API 适配层。
3. 当前阶段通过 `/app/tasks?taskId=<id>` 预留跨页面联动入口，后续聊天页和历史页可直接复用该入口打开详情抽屉。
