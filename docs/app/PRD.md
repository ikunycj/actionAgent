## 2. 模块定位

Native Client 是 ActionAgent 的主要用户入口。

它运行在用户自己的设备上，负责承接：

1. 日常对话
2. 历史浏览
3. 任务操作
4. 远程或本地 Core 连接

客户端不替代 Core，也不替代 WebUI 管理台；它负责高频使用场景，WebUI 负责高级管理场景。

## 3. 模块目标

1. 为用户提供比浏览器更稳定的主使用入口
2. 把日常高频交互从 WebUI 转移到客户端
3. 在不改变 Core 事实源的前提下，承接主要交互能力
4. 在必要时跳转到对应 Core 的 WebUI 管理台，并可携带 agent 级筛选上下文

## 4. 功能范围

### 4.1 第一阶段目标

1. 连接用户自己的 Agent Core
2. 完成认证和会话恢复
3. 承载主要对话入口
4. 查看历史和任务
5. 展示当前连接的 Core 与运行范围信息

### 4.2 第二阶段目标

1. 管理多个 Core 连接
2. 提供本地通知、托盘、后台常驻等能力
3. 打开对应 Core 的 WebUI 管理页，并在需要时定位到对应 agent 过滤视图
4. 管理本地独立 Core 实例

## 5. 设计约束

1. 客户端是主入口，不是附属工具
2. 客户端对配置的修改若涉及高级管理，可跳转到 WebUI
3. Core 的配置和数据边界仍由 Core 控制
4. 本地原生能力不得直接暴露给远程 Core

## 6. 非目标

1. iOS 首发交付
2. 把客户端做成远程 Core 的设备能力代理
3. 先于 Core 和 WebUI 完成全部产品能力
4. 用客户端替代 WebUI 管理台的高级配置能力

## 7. 后续演进

1. 先完成连接、认证、对话、历史、任务主链路
2. 再补托盘、通知和本地 Core 管理
3. 再做多 Core 和更强的管理能力
