## 2. 模块定位

Agent Core 是产品的执行面、配置事实源和控制面宿主。

它不再只是一个纯 API 进程，还承担两项产品职责：

1. 提供稳定 API 给客户端、自动化调用和控制台
2. 在生产发布中提供与 Core 同包交付的 WebUI 控制台

## 3. 模块目标

1. 以单进程或单发布包交付“API + WebUI 控制台”。
2. 让控制台默认走本地安全路径，而不是先暴露远程管理面。
3. 保持客户端可稳定接入的 API 边界。
4. 让 WebUI 明确围绕单个 Agent 的配置、治理和调试工作。
5. 把审批、审计和密钥保护纳入控制面主链路。

## 4. 角色关系

### 4.1 Core

1. 执行任务和模型调用。
2. 保存配置与密钥引用。
3. 提供 HTTP、Bridge、事件和观测接口。
4. 提供控制面认证、授权、审批和审计能力。
5. 提供 WebUI 静态资源。

### 4.2 Bundled WebUI

1. 管理单个 Agent。
2. 完成配置、状态、历史、任务和诊断操作。
3. 承载配对、会话恢复和高危操作确认。

### 4.3 Native Client

1. 作为主要用户入口连接 Core。
2. 使用同一套核心 API 完成日常交互。
3. 在必要时跳转到对应 Agent 的控制台。

## 5. 功能范围

### 5.1 已有核心运行能力

1. `GET /healthz`
2. `POST /v1/run`
3. `POST /v1/chat/completions`
4. `POST /v1/responses`
5. `POST /ws/frame`
6. `GET /events`
7. `GET /metrics`
8. `GET /alerts`

### 5.2 控制面目标能力

1. 控制台静态资源托管
2. 控制面认证与会话恢复
3. 作用域化授权
4. 配置读写、应用和回滚
5. 审批、审计和密钥保护
6. 按 Agent 的状态、历史和任务查询

## 6. 非目标

1. 把 Core 做成多租户 SaaS 宿主。
2. 让 WebUI 继续以独立部署产品为正式路径。
3. 把 WebUI 做成多 Agent 编排平台级总控。
4. 让浏览器端直接保存长期高权限 Token 成为正式默认方案。

## 7. 核心需求

### 7.1 同包控制台交付

目标：启动 Core 后即可访问控制台。

1. Core 必须能提供 WebUI 静态资源。
2. API 路由与 WebUI 路由必须共存，不得互相覆盖。
3. 生产主路径不应要求用户再部署前端服务。

### 7.2 控制面安全默认值

目标：默认路径先满足本地安全，再显式支持远程治理。

1. Core 默认监听 loopback。
2. loopback 场景默认采用 `local-paired` 控制面访问模式。
3. 非 loopback 暴露时，必须显式启用 `token` 或 `trusted-proxy` 等远程安全模式。
4. 非 loopback 暴露不得在无认证模式下启动控制面。

### 7.3 会话与鉴权模型

目标：让浏览器控制台和自动化调用使用适合自己的安全模式。

1. 浏览器控制台默认使用 Cookie 会话，而不是长期前端存储 Token。
2. 浏览器写操作必须具备 CSRF 防护。
3. CLI、脚本或远程自动化调用可使用带 scope 的 Bearer Token。
4. 鉴权上下文必须包含 subject、device/session、agent scope 和 capability scope。

### 7.4 Agent 作用域与能力作用域

目标：控制台的“单 Agent 管理”不仅是 UI 约束，也是后端权限约束。

1. 每次控制面请求必须解析目标 Agent。
2. 未显式指定时，可回退到 `default_agent`。
3. 后端必须按 `agent_id` 与 capability scope 共同判定授权。
4. 读权限、执行权限、配置写权限、密钥写权限、审批权限和运行时管理权限必须可区分。

### 7.5 高危操作审批与审计

目标：高危动作具备二次保护，而不是“登录后全能”。

1. 配置应用、密钥写入、暴露面放宽和高危工具策略变更必须走审批。
2. 审批必须绑定上下文，不能脱离目标 Agent 和动作复用。
3. 工具执行、审批决策、配置变更和控制面认证事件必须可审计。
4. 审计导出必须支持脱敏。

### 7.6 密钥处理

目标：控制台可以管理密钥，但不能把密钥当普通配置字段读回。

1. 密钥写入后不得通过普通读接口明文返回。
2. 界面仅展示是否已设置、最后更新时间和必要的元信息。
3. 日志、审计和错误消息不得泄漏原始密钥内容。

## 8. 验收标准

1. Core 启动后可直接访问控制台。
2. `/v1/*`、`/ws/frame` 与静态资源路由稳定共存。
3. 控制台可围绕某个 Agent 完成配置、历史和任务操作。
4. 控制面具备认证、作用域校验、审批和审计闭环。
5. 用户无需再单独部署 WebUI，也无需依赖默认账号密码启动控制面。
